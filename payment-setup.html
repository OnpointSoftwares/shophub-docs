Payment Setup Guide

Configure M-Pesa payment integration using PayHero or Safaricom Daraja API.

Overview

ShopHub supports two M-Pesa payment gateways:

PayHero

Third-party payment gateway with simplified integration

  • Easy setup process
  • Managed infrastructure
  • Built-in callback handling
  • Transaction fees apply
Recommended for Beginners

Safaricom Daraja API

Direct integration with Safaricom M-Pesa

  • Direct M-Pesa integration
  • Lower transaction fees
  • More control over flow
  • Complex setup process

PayHero Setup

Current Status: If you see "Merchant Account Inactive" error, your PayHero account needs activation. Use Daraja API in the meantime.
1Create PayHero Account
  1. Visit https://payhero.co.ke
  2. Click "Sign Up" and create an account
  3. Complete business verification (KYC)
  4. Wait for account approval (1-2 business days)
2Get API Credentials
  1. Login to PayHero dashboard
  2. Navigate to Settings → API Keys
  3. Copy your API Key (Base64 encoded)
  4. Copy your Channel ID
3Configure Environment Variables

Add the following to your .env file:

# PayHero Configuration
PAYHERO_API_KEY=Basic YOUR_BASE64_API_KEY_HERE
PAYHERO_CHANNEL_ID=your_channel_id
PAYHERO_CALLBACK_URL=https://yourdomain.com/api/payhero/callback
Important: The API key must include the "Basic " prefix!
4Test Integration
  1. Start your server: npm start
  2. Navigate to checkout page
  3. Select "PayHero" as payment provider
  4. Enter test phone number
  5. Complete test payment

Safaricom Daraja API Setup

Recommended: Use this option while waiting for PayHero account activation.
1Create Safaricom Developer Account
  1. Visit Safaricom Developer Portal
  2. Click "Sign Up" and create an account
  3. Verify your email address
  4. Complete your profile
2Create an App
  1. Login to developer portal
  2. Click "My Apps" → "Create New App"
  3. Select "Lipa Na M-Pesa Online" API
  4. Fill in app details and submit
3Get Credentials

After app creation, you'll receive:

  • Consumer Key
  • Consumer Secret
  • Shortcode (for sandbox: 174379)
  • Passkey
4Configure Environment Variables
# M-Pesa Daraja Configuration
MPESA_ENVIRONMENT=sandbox
MPESA_CONSUMER_KEY=your_consumer_key_here
MPESA_CONSUMER_SECRET=your_consumer_secret_here
MPESA_SHORTCODE=174379
MPESA_PASSKEY=your_passkey_here
MPESA_CALLBACK_URL=https://yourdomain.com/api/payment/mpesa/callback
5Set Up Callback URL

For local testing, use ngrok:

# Install ngrok
npm install -g ngrok

# Start your server
npm start

# In another terminal, start ngrok
ngrok http 3000

# Copy the HTTPS URL and update MPESA_CALLBACK_URL
6Go Live
Production Requirements

Troubleshooting

Solution:
  • Contact PayHero support for account activation
  • Complete all KYC requirements
  • Use Daraja API as alternative

Solutions:
  • Verify phone number format (0712345678 or 254712345678)
  • Ensure phone number is Safaricom
  • Check phone has network coverage
  • Verify API credentials are correct

Solutions:
  • Ensure callback URL is publicly accessible
  • Use ngrok for local testing
  • Check server logs for incoming requests

Security Best Practices
Critical Security Measures
  • Never commit .env file to version control
  • Use environment variables for all credentials
  • Always use HTTPS in production
  • Validate all callback data
  • Implement rate limiting on payment endpoints
  • Monitor transactions for anomalies

Quick Reference

PayHero Endpoints
  • POST /api/payhero/initiate - Initiate payment
  • GET /api/payhero/status/:orderId - Check status
  • POST /api/payhero/callback - Webhook
Daraja Endpoints
  • POST /api/payment/mpesa/initiate - Initiate payment
  • GET /api/payment/mpesa/status/:orderId - Check status
  • POST /api/payment/mpesa/callback - Webhook
